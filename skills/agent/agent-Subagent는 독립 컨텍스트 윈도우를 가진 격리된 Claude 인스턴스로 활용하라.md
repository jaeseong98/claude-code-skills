# Subagent는 독립 컨텍스트 윈도우를 가진 격리된 Claude 인스턴스로 활용하라

Subagent는 메인 대화와 별도의 컨텍스트 윈도우에서 실행되며, 태스크 완료 후 결과만 메인 대화로 반환한다. 수십 개 파일 탐색 리서치, 독립적 병렬 수정, 편견 없는 코드 검증, 단계별 파이프라인 워크플로우에 적합하다.

---

## Usage

# Subagent 활용 판단 가이드

## 언제 사용하나
- 수십 개 파일을 탐색하는 리서치 작업
- 서로 독립적인 코드 수정을 병렬로 처리할 때
- 기존 대화 맥락에 편향되지 않은 코드 검증이 필요할 때
- 단계별 파이프라인 워크플로우 구성 시

## Gotchas
- Subagent는 메인 대화의 컨텍스트를 공유하지 않으므로 필요한 정보를 프롬프트에 명시해야 한다
- 결과만 반환되므로 중간 과정이 필요하면 로깅을 별도로 설계하라
